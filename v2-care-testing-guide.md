# V2-Care 功能体验指南

> 你已经设置了 API Key，且有 10 天历史记录。
> 本指南帮你逐一验证四个 AI 功能是否正常工作。

---

## 前置检查

打开 V2-Care，先确认：

1. 点击右上角 **⚙️** 图标
2. 确认 API Key 已保存（显示 `✅ DeepSeek API 已配置`）
3. 如果没有，重新输入 API Key 并点击「保存」

---

## 功能一：鼓励语个性化 ✦

### 如何触发
提交一条感恩条目后，鼓励区（首页底部淡绿色文字）会显示一句鼓励语。

### 如何确认 AI 在工作
- **普通 V2**：鼓励语从固定数组随机选，可能重复
- **V2-Care**：鼓励语由 AI 根据以下内容**实时生成**：
  - 你今天写的三件感恩
  - 你的历史记录（总条数、最近写了什么）
  - 当前里程碑（首次/连续3天/共7次/共20次）

**验证方法**：
连续提交 3-5 次，观察鼓励语是否：
- 每次都不同（不是固定数组里的句子）
- 内容与你今天写的内容有关联（比如你写了"朋友请吃饭"，鼓励语提到"他人的温暖"）

### 如果看不到差异
- 首次提交可能仍使用默认鼓励语（里程碑 M1/M2 用模态框，不用鼓励区）
- 从第 3 次提交开始，鼓励区出现，AI 鼓励语才会显示

## 功能二：语义 Reunion 匹配 ✦

### 如何触发
1. 提交感恩条目后，首页右下角会出现 **「✦ 重逢」** 水印
2. 点击水印，或等待自动触发（随机延迟 3-8 秒）
3. 进入 Reunion 全屏，显示一条**历史上的感恩**

### 如何确认 AI 在工作
- **普通 V2**：从历史记录中**完全随机**选一条
- **V2-Care**：AI 会分析你**今天写的内容**，从历史上找**语义最相关**的一条

**验证方法**：
1. 今天写一条特定主题的感恩（比如「今天同事帮我debug」）
2. 提交后触发 Reunion
3. 看回来的那条旧感恩，是否与「工作/帮助/同事」相关

**举例**：
- 今天写：「感谢同事帮我解决了bug」
- 普通V2可能返回：「感谢今天的阳光很暖」（完全随机）
- V2-Care应该返回：「感谢上次pair programming学到了很多」（语义相关）

### 如果看不到差异
- 历史记录太少（<5条）时，语义匹配空间有限，可能看起来还是随机的
- 你有10天记录，应该已经能感知到差异

## 功能三：「✦ 给今天的我」洞察 ✦

### 如何触发
- 在 Reunion 全屏中，除了显示旧感恩，有时会在卡片下方出现一段**额外文字**
- 格式：「✦ 给今天的我：……」
- 这是 AI 根据旧感恩和今天的内容，生成的一段**回响/洞察**

### 如何确认 AI 在工作
- **普通 V2**：没有这个功能
- **V2-Care**：有概率出现（不是每次都有，保持惊喜感）

**验证方法**：
1. 连续触发 Reunion 3-5 次（可以手动点击水印）
2. 观察是否出现「✦ 给今天的我：……」文字
3. 这段文字应该是**连接过去和现在**的，不是通用鸡汤

### 如果看不到
- 这是**概率触发**的（约30%概率），不是每次都有
- 需要 AI 调用成功 + 生成内容通过质量检测
- 连续试几次就能看到

## 功能四：签名选词情感匹配 ✦

### 如何触发
每次提交感恩后，卡片底部左下角会显示一个**英文签名词**（peaceful / love / soul / ... 共30个）

### 如何确认 AI 在工作
- **普通 V2**：完全随机选词
- **V2-Care**：AI 会根据你今天写的内容**选择最匹配情绪的词**

**验证方法**：
1. 写一条特定情绪的内容，比如：
   - 写了开心的事 → 签名词应该是 joyful / happy / bright 类
   - 写了平静的事 → 签名词应该是 peaceful / calm / quiet 类
   - 写了感动的事 → 签名词应该是 grateful / warmth / tender 类
2. 连续试 3-5 次，观察签名词是否与内容情绪匹配

### 如果看不到差异
- 前几次可能还是随机（AI 调用需要时间，首次可能降级到随机）
- 签名词变化比较微妙，需要刻意观察才能感知

## 如果某个功能不工作

### 检查清单
1. **API Key 是否有效**：在设置面板点击「测试连接」按钮（如有），或提交一条记录看控制台是否有错误
2. **网络是否正常**：AI 调用需要访问 `api.deepseek.com`，确认网络可访问
3. **打开浏览器控制台**（F12），看是否有红色错误信息

### 常见错误
- `API Key 无效` → Key 填错了，或余额不足
- `API error 429` → 调用频率超限，等待1分钟后重试
- `AI 调用超时` → 网络问题，或 DeepSeek 服务暂时不可用

### 降级行为
所有 AI 功能都有**优雅降级**：
- AI 调用失败 → 自动使用 V2 原版逻辑（随机鼓励语/随机Reunion/随机签名词）
- 所以即使 AI 不工作，V2-Care 也**不会报错**，只是表现和 V2 一样
- 这也是你之前感觉"完全没区别"的原因

## 最佳实践：如何快速体验所有功能

### 5分钟快速验证
1. 打开 V2-Care，确认 API Key 已配置
2. 写一条感恩：「感谢今天和老朋友聊了很久，收获很多」
3. 提交后，观察鼓励区文字（应该和"朋友/交流"相关）
4. 点击 Reunion 水印，看返回的旧感恩是否与"人际/交流"相关
5. 观察卡片签名词，是否是 warmth / grateful / connection 类

### 15分钟深度验证
1. 连续提交 3-5 条不同主题的感恩（工作/家庭/自然/自我/他人）
2. 每条都触发 Reunion，记录返回的旧感恩主题
3. 对比：V2-Care 的 Reunion 是否比随机更有意义？
4. 观察鼓励语是否有变化
5. 观察签名词是否随内容情绪变化
